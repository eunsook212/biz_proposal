01 / Overview · 과정 소개

‘도구 1개’가 아니라,
‘단계별 도구 배치’를 배웁니다.

바이브 코딩의 기본기와 최신 도구들의 활용법을 익히되, 단순 사용법 학습에서 멈추지 않습니다. 설계는 어떤 도구로, 빌드는 어떤 도구로, 다듬기는 어떤 도구로 할 것인가 — 각 단계마다 가장 빛나는 도구를 배치하는 실전 워크플로우를 구축합니다.

A

기획 직무 맞춤 설계

비개발자가 코드 한 줄 없이 실제 동작하는 AI 웹앱을 만들고 배포할 수 있도록, 기획에서 빌드, 디버깅, 배포까지 모든 단계를 자연어 지시만으로 처리하는 흐름을 다룹니다.

B

도구별 강점의 전략적 활용

Google AI Studio의 즉각성, Codex Desktop의 빌드 능력, Claude Code의 다듬기 정밀도 — 각 도구가 어느 국면에서 가장 효과적인지를 식별하고, 본인 업무에 적용하는 안목을 기릅니다.

02 / Goals · 교육 목표

교육이 끝날 때,
이 4가지가 손에 남습니다.

학습 목표는 단순합니다. 8시간 후 참가자가 혼자서도 이 흐름을 다시 돌릴 수 있는 상태가 되는 것입니다.

01

AI 웹앱의 구조를 이해한다

프론트엔드 · 백엔드 · 데이터베이스 · API · RAG, 5대 기술 레이어의 역할과 연동 구조를 비개발자 관점에서 명확히 이해하고, 도구가 어느 레이어를 자동화하는지를 식별합니다.

02

3가지 바이브 코딩 도구를 능숙하게 다룬다

Google AI Studio로 즉석 프로토타입을, Codex Desktop으로 본격 빌드를, Claude Code로 디버깅과 리팩토링을 — 각 도구가 가장 빛나는 단계에 올바르게 배치하는 감각을 익힙니다.

03

기획에서 배포까지 전 과정을 수행한다

리서치와 기획서 초안 자동화부터, 멀티파일 기능 구현, 디버깅과 코드 다듬기, 그리고 Vercel 배포로 공개 URL 확보까지 — 한 사이클을 본인의 손으로 완주합니다.

04

본인 업무에 적용할 첫 액션 아이템을 도출한다

교육장에서 끝나지 않습니다. 본인 업무에서 당장 시도해볼 수 있는 자동화 아이디어 1건과, 사내 확산을 위한 다음 행동을 정리하여 가져갑니다.

03 / Stack · 핵심 기술 스택

도구 3개와 배포 1개,
각자의 자리가 있습니다.

모든 도구가 모든 단계에 적합하지 않습니다. 본 과정은 각 도구가 가장 효과적인 국면에서만 사용되도록 동선을 설계합니다.

Google AI Studio
QUICK PROTOTYPE

브라우저에서 자연어 프롬프트만으로 풀스택 웹앱을 즉시 생성. 첫 진입과 빠른 검증에 사용합니다.

Codex Desktop
PLAN & BUILD

데스크톱 기반 에이전트. 시장 리서치·기획 초안 자동화부터 멀티파일 빌드까지 한 흐름으로 처리합니다.

Claude Code
REFINE & ITERATE

터미널 기반 에이전트. 오류 디버깅·코드 다듬기·반복 개선으로 결과물의 완성도를 끌어올립니다.

Vercel
DEPLOY

만든 웹앱을 공개 URL로 즉시 배포. 누구나 접속 가능한 실제 서비스 형태로 결과를 공유합니다.

04 / Infrastructure · 필요 인프라

준비물은 최소한입니다.

별도의 개발 환경 세팅은 필요하지 않습니다. 노트북 한 대와 Google 계정만 있으면 됩니다.

외부 교육장 (별도 장소 운영)
노트북 1인 1대 (렌탈 또는 개인 지참)
외부망 Wi-Fi 접속 환경
참가자별 Google 계정
Codex / Claude Code API 환경 (사전 제공)
Vercel 계정 (현장 가입 가능)
05 / Curriculum · 상세 커리큘럼

8시간 안에
한 사이클을 완주합니다.

7개 모듈, 총 480분. 개념 정렬부터 시작해 즉석 프로토타입, 기획 자동화, 본격 빌드, 완성도 다듬기, 배포, 사내 확산 가이드까지 — 도구를 단계별로 갈아타며 같은 아이디어를 완성시킵니다.

STAGE 1 · 개념 정렬

바이브 코딩의 작동 원리를 한 그림으로

왜 비개발자도 만들 수 있게 되었는가, 그리고 무엇을 알아야 효과적으로 만드는가

  • 바이브 코딩(Vibe Coding)이란? 자연어로 코드를 만드는 시대의 트렌드와 한계, 그리고 가능성의 경계선
  • AI 웹앱의 5대 기술 레이어 해설 — 프론트엔드(사용자 화면)·백엔드(비즈니스 로직)·데이터베이스(데이터 저장)·API(프로그램 간 소통)·RAG(AI 지식 주입)의 역할과 연동 구조
  • 도구 배치 전략 개요 — 왜 한 도구로 다 하지 않는가, 단계별로 도구를 갈아타는 것이 효과적인 이유
  • 비개발자가 알아야 할 최소한의 개념과, 알 필요 없는 것의 경계

코딩 에이전트를 기획 파트너로

  • 실습기획서 초안 제작 — 양식에 맞춘 기능 정의·화면 구조·데이터 구조 초안 작성 자동화
  • 실습기획안을 빌드 프롬프트로 변환 — 완성된 기획을 Codex가 즉시 구현할 수 있는 빌드용 프롬프트로 정제하는 기술

Codex Desktop
STAGE 4 · 본격 빌드

기획안을 실제 웹앱으로 구현한다

Codex의 진짜 강점: 컨텍스트를 유지하며 여러 파일을 동시에 만들어내는 능력

  • 강사 시연 — Codex Desktop으로 웹앱 하나를 기획부터 구현까지 단계별 전체 흐름 시연
  • 실습프로젝트 스캐폴딩 — 자연어 요구사항 하나로 폴더 구조·설정 파일·보일러플레이트를 한 번에 생성
  • 실습멀티 파일 코드 생성 — 기능 명세를 입력하면 연관된 여러 파일(화면·로직·데이터)을 컨텍스트를 유지하며 동시에 작성하는 워크플로우
  • 실습개별 웹앱 빌드 — 본인의 기획 아이디어를 기반으로 개인별 AI 웹앱 구현
  • 중간 점검과 피드백 — 강사·보조강사가 빌드 진도와 결과물을 즉시 점검하고 개선 방향 제시

STAGE 5 · 완성도 다듬기

만든 것을 다듬어 완성도를 높인다

Claude Code의 진짜 강점: 터미널에서 직접 오류를 읽고, 분석하고, 수정하는 자율성

  • Codex와 Claude Code의 역할 분담 — 왜 빌드는 Codex로, 다듬기는 Claude Code로 가는 것이 효과적인가
  • 실습터미널 기반 디버깅 — 빌드 에러·런타임 오류 발생 시 Claude Code가 로그를 직접 읽고 원인을 분석하고 파일을 수정하는 일련의 과정을 터미널에서 실행
  • 실습코드 다듬기 — 기존 코드베이스를 Claude Code에게 분석시키고, 중복 제거·네이밍 개선·구조 정리를 지시하여 코드 품질을 단계적으로 높이는 워크플로우
  • 실습기능 추가와 반복 개선 — 자연어 지시만으로 추가 기능을 붙이고, 즉시 결과를 확인하며 반복 개선하는 사이클

  • 실습결과물 공유 — 팀별 또는 전체 공유 화면에서 본인의 URL과 작동 시연 발표
  • 상호 피드백 — 다른 참가자의 결과물을 직접 접속해 사용해보고 개선 아이디어 교환

06 / Deliverables · 예상 산출물

이론 자료가 아니라,
실물을 가져갑니다.

교육 종료 시점, 모든 참가자는 본인의 손으로 만든 다음 3가지 산출물을 보유하게 됩니다.

DELIVERABLE 01

Google AI Studio 기반 AI 웹앱 프로토타입

브라우저에서 즉석 생성한 첫 AI 웹앱 1건. 아이디어 검증용 프로토타입으로 즉시 활용 가능합니다.

DELIVERABLE 02

Codex 기반 기획 자동화 산출물

Codex Desktop으로 자동 생성한 리서치 결과 + 기획 초안 패키지 1건. 본인 업무 기획에도 그대로 응용 가능합니다.

DELIVERABLE 03

완성된 AI 웹앱 (Vercel 배포 URL)

Codex로 빌드하고 Claude Code로 다듬어 Vercel에 배포한 공개 접속 가능한 웹앱 1건. 사내 공유와 즉시 시연 가능합니다.

07 / NOTES & CLOSING

‘안다’에서 멈추지 않고,
‘만들어 봤다’까지 갑니다.

본 과정은 도구의 사용법을 가르치는 것이 아니라, 도구를 단계별로 올바르게 배치하는 안목을 만드는 것을 목표로 합니다. 교육이 끝날 때 손에 남는 것은 학습 자료가 아니라, 본인의 손으로 만들어 배포한 실제 웹앱입니다.

비고
  • 본 과정은 총 8시간(1일)으로 편성되며, 운영 시작·종료 시각은 조직 일정에 맞춰 협의 가능합니다.
  • 비개발 직군 대상 과정으로, CLI 환경이나 프로그래밍 사전 지식은 필요하지 않습니다. 모든 도구는 자연어 지시로 운영됩니다.
  • 참가자 규모에 따라 보조 강사 운영을 통해 실습 진도와 개인별 빌드 결과물을 함께 관리합니다.
  • 커리큘럼은 조직의 업무 맥락에 맞춰 사전 협의를 통해 일부 조정 가능합니다.
  • 본 과정 종료 후 후속 심화 과정(에이전틱 코딩 심화 · 데이터 활용 · 멀티에이전트 오케스트레이션 등) 연계 운영도 가능합니다.
